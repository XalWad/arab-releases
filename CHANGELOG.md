# ARAB — Changelog

## v1.0.9 (2026-06-30)

Лента активности, нумерация аккаунтов, фикс окна подтверждения.

### Новое
- **Консоль логов внизу приложения (как в клиенте MoonTrader).** Нижняя строка статуса
  теперь показывает последнюю запись лога + счётчик предупреждений/ошибок; по клику
  **раскрывается вверх в консоль** — список событий с временем, фильтр, счётчики warn/error,
  «Очистить» и «Папка логов». Пишутся все уровни (действия, инфо, предупреждения, ошибки):
  опрос статуса серверов, управление ядром, ошибки связи и т.д. — видно, что программа
  работает, а не зависла, и можно найти ошибку.
- **Порядковые номера аккаунтов** — слева от карточек мелкая полупрозрачная нумерация
  (статичная: показывает позицию в списке, при перемещении аккаунтов номера остаются на
  местах).
- **Поиск/фильтр аккаунтов** — поле с лупой 🔍 в шапке (рядом с «+ Добавить»): вводишь имя,
  папку MT или IP — список показывает только совпадения (счётчик «N из всего»). Удобно на
  большом парке аккаунтов.
- **Копирование алгоритмов и папок в формате MoonTrader** — кнопка «⧉» у каждого алгоритма
  и каждой папки (во вкладках «Алгоритмы» и «Шаблоны»). Копирует в буфер в текстовом формате
  MT — можно **вставить прямо в MoonTrader**.
- **Экспорт в текст MoonTrader (.txt)** — во вкладке «Алгоритмы» кнопка «⭳ Экспорт» (все
  алгоритмы аккаунта), во вкладке «Шаблоны» — «↓ Экспорт» (активный шаблон). Файл в формате
  MT, пригоден для вставки в MoonTrader.
- **Массовое управление ядром** — добавлен второй тип перезагрузки: теперь «🔄 Перезапуск
  (no-update)» и «🔄 Перезапуск (update)» (как в меню карточки), а не один вариант.

### Исправления
- **Окно подтверждения массового управления ядром больше не прячется под основным окном.**
  При «Применить к N» подтверждение рисовалось ПОД окном (фон затемнялся, но диалога не
  видно). Теперь оно поверх.

## v1.0.8 (2026-06-30)

Проверка связи с сервером, авто-восстановление ядра после деплоя, фикс прокрутки окна обновления.

### Новое
- **Кнопка «🔌 Проверить связь» в форме аккаунта.** По введённым адресу и секрету проверяет
  доступность сервера и сразу говорит, ГДЕ проблема, если связи нет:
  - ✅ порт открыт + Core отвечает (показывает версию);
  - 🔑 порт открыт, но секрет неверный;
  - 💥 порт открыт, но Core не отвечает (лежит/перезапуск);
  - 🔌 порт закрыт — Core не слушает (ядро лежит / не тот порт);
  - ⛔ таймаут — порт режет фаервол (AWS Security Group / хостинг), надо открыть 4343/tcp;
  - ❓ хост/IP не резолвится.
  Под капотом — сырой TCP-коннект из main-процесса (различает «закрыто» от «таймаута», чего
  WebSocket в браузере не умеет) + WS-рукопожатие для проверки секрета и версии.
- **Авто-восстановление ядра MT после деплоя/рестарта (на стороне ядра).** Иногда после
  деплоя MTCore делает «чистое завершение», которое MTGuardian трактует как остановку
  пользователем — уходит в режим мониторинга и НЕ поднимает ядро обратно. Раньше помогало
  только ручное «Включение». Теперь ARAB Core ~2 минуты после операции, которая должна
  оставить ядро живым (деплой / рестарт / откат), следит за ядром и, если оно легло, **сам
  поднимает его** один раз (через mt_start). Осознанное «Выключение» из меню при этом
  уважается — его watchdog не воскрешает.

### Исправления
- **Перезапуск ядра MT переделан и больше не «роняет» ядро.** Раньше рестарт делал
  `pkill MTCore`, оставляя MTGuardian живым → гонка (два ядра, одно само выключалось) или
  уход Guardian в monitoring без подъёма ядра. Теперь оба рестарта — это чёткая
  последовательность **легальный стоп → пауза → старт в том же screen** (Guardian остаётся
  следить, без гонки). «Перезапуск (update)» при этом стартует `./MTCore` без `--no-update`,
  «no-update» — с ним.
- **Окно «Доступно обновление» теперь прокручивается.** При большом числе серверов список
  Core не помещался на экран, кнопка «⬆ Обновить выбранное» уезжала за низ и до неё нельзя
  было добраться. Тело окна (список серверов + changelog) сделано прокручиваемым, заголовок
  и кнопки зафиксированы.
- **Редактирование аккаунта открывается под самой карточкой**, а не сверху списка (форма
  добавления нового — по-прежнему сверху, как и логично).

## v1.0.7 (2026-06-29)

Режим приватности, переработка панели аккаунтов и доводка экспорта CSV.

### Новое
- **Замок приватности 🔒 теперь скрывает и состояние серверов.** Раньше прятались только
  IP и папка. Теперь в режиме приватности (для скриншотов/стримов) скрываются также биржа,
  версия Core, статус ядра MT и нагрузка (RAM/CPU/Disk), а строки аккаунтов становятся
  компактнее — остаётся имя + скрытые адрес/секрет.
- **Импорт и экспорт CSV объединены в одну кнопку «🗂 CSV».** По нажатию открывается окно
  с выбором «Экспорт» / «Импорт»; подсказка по формату CSV теперь внутри этого окна (убрана
  отдельная раскрывашка с вкладки настроек).
- **«Telegram chat ID для всех» → кнопка с окном.** Вместо раскрывающегося поля — кнопка,
  открывающая попап, где задаётся общий chat ID с применением ко всем аккаунтам. Значок
  Telegram (в карточке и на кнопке) заменён на бумажный самолётик.
- **Кнопка ребута переименована в «Ребут всех серверов»** и теперь красная по умолчанию
  (красный значок ↻, текст и выделение) — подчёркивает ответственность действия. У кнопки
  «Статус серверов» значок сменён на 🩺 (здоровье серверов).
- **Массовое управление ядром MT — кнопка «⚙ Ядро» в шапке.** Открывает окно: выбираешь
  действие (▶ Включить / 🔄 Перезапустить / ⏹ Выключить) и аккаунты по галочкам — с выбором
  всех сразу и по группам-биржам — затем применяешь разом (с прогрессом по каждому).
- **Действия с ядром в карточке аккаунта объединены в одну кнопку «🔧».** По нажатию —
  меню с цветными действиями: ▶ Включение (зелёный), ⏹ Выключение (оранжевый), 🔄 Перезапуск
  no-update (жёлтый) и 🔄 Перезапуск с **обновлением** ядра MT (жёлтый).
- **Окна подтверждения — в едином стиле приложения.** Подтверждения ребута сервера(ов),
  действий с ядром и удаления аккаунта больше не используют системный диалог Windows —
  теперь это аккуратные тёмные окна в общей стилистике (Enter/Esc работают).

> ⚙ «Перезапуск с обновлением» — это изменение и в ядре (новый `restart_mt_update.sh`).
> Чтобы оно работало, после релиза обнови Core на серверах (кнопкой ⬆). Поведение
> обновления MTCore зависит от MTGuardian — проверь на одном сервере.

### Исправления
- **Заголовок файла экспорта CSV — на английском + UTF-8 BOM.** Служебная первая строка
  открывалась «кракозябрами» в редакторе под Windows-1251. Теперь файл начинается с BOM
  (корректно читается в Excel/блокноте), а комментарий полностью на английском.

## v1.0.6 (2026-06-29)

Критфикс самообновления Terminal + экспорт аккаунтов и мониторинг диска.

### Новое
- **Экспорт аккаунтов в CSV** — кнопка «📤 Экспорт CSV» рядом с импортом. Сохраняет все
  аккаунты в том же формате (`Name,IP,Secret,Exchange`), что принимает импорт, — удобно
  перенести список на другой ПК и импортировать обратно. ⚠️ Файл содержит секреты — хранить
  безопасно.
- **Загрузка диска (SSD) в карточке аккаунта** — рядом с RAM и CPU теперь показывается
  заполненность корневого раздела: `💽 Disk 80% (8.00/10.00 ГБ)`. Подсветка оранжевым при
  ≥85%. Данные отдаёт ядро в снимке нагрузки (нужно обновить Core на серверах).

### Исправления
- **Кнопка обновления Terminal больше не выдаёт «Please check update first».** При нажатии
  «⬆ Обновить выбранное» клиент сразу качал апдейт, не сделав обязательный для
  electron-updater предварительный `checkForUpdates()` — отсюда ошибка. Теперь загрузка
  сначала проверяет наличие новой версии и только потом качает.
- **Старый клиент снова видит новые версии.** Фид обновлений был привязан к собственной
  версии сборки (url `.../releases/download/v${version}` вшивался в `app-update.yml` при
  сборке), поэтому, например, клиент 1.0.3 навсегда смотрел в папку релиза `v1.0.3` и
  физически не мог обнаружить более новые. Перевели фид на github-провайдер — он находит
  самый свежий релиз в репозитории независимо от версии клиента.

> ⚠️ Установленные старые клиенты (1.0.3–1.0.5) этот фикс «по воздуху» получить не могут —
> сломанный механизм обновления вшит в их бинарь. Их нужно **один раз обновить вручную**
> (скачать и установить Setup.exe этой версии). Начиная с 1.0.6 кнопка обновления работает
> штатно для всех будущих версий.

## v1.0.5 (2026-06-25)

Управление ядром MoonTrader из терминала + критфикс деплоя.

### Новое
- **Стоп / Рестарт / Старт ядра MoonTrader** — в карточке аккаунта три кнопки:
  **⏹ Стоп** (легальная остановка через Ctrl+C — MTGuardian считает это нормальным
  отключением и НЕ поднимает ядро обратно), **🔄 Рестарт** (как раньше), **▶ Старт**
  (запуск ядра; кнопка активна только если ядро выключено). Всё по имени процесса, без
  хардкода PID — работает на любом сервере.
- **Статус ядра в карточке аккаунта** — бейдж **🟢 MT** (активно) / **🔴 MT** (остановлено).

### Исправления
- **Деплой больше не роняет MTCore из-за «пустой папки».** Если в шаблоне/конфиге была
  папка без типа (`groupType:0` — такой у новой папки, пока в неё не добавлен алгоритм)
  или алгоритм «без папки», в MoonTrader уезжала невалидная структура — MTCore падал на
  старте, и даже MTGuardian не мог поднять ядро (бесконечное «No MTCore process running»).
  Теперь и ядро (при записи конфига), и Terminal (перед деплоем) **чинят структуру**:
  алгоритм-сирота попадает в подходящую папку, папке без типа проставляется тип по её
  алгоритму, пустые папки без типа отбрасываются. Кривой конфиг физически не уходит в MT.

> Управление ядром, статус и защита деплоя живут **в самом ядре** — обнови Core на серверах
> (кнопкой ⬆), и они заработают для всех аккаунтов независимо от версии Terminal.

## v1.0.4 (2026-06-25)

Имена алгоритмов в стиле MoonTrader, отложенные изменения и доработки UI.

### Новое
- **Компактнее окно редактора алгоритма** — блок имени перенесён в прокручиваемую панель
  параметров (не занимает фиксированное место сверху), панель фильтров слева растянута на
  освободившуюся высоту, шапка-название сделана тоньше.
- **Имена алгоритмов в стиле MoonTrader (шаблон с кодами).** В редакторе алгоритма (сверху
  панели параметров) — поле шаблона с кодами (`##SIDE##`, `##DIST##`, `##TP##`, `##BUFF##`, `##QVOL##`…).
  Печатать коды не нужно: кликаешь по кнопкам-кодам, они вставляются в шаблон, а строкой
  ниже сразу видно итоговое имя, которое покажет MoonTrader. Набор кодов зависит от типа
  алгоритма (MoonShot/DepthShot). MT сам пересобирает имя по шаблону при изменении параметров.
- **Отложенные изменения (без мгновенного деплоя).** Действия во вкладке «Алгоритмы»
  (правка параметров, создание/удаление/перемещение, импорт шаблона) теперь копятся в
  рабочей копии и НЕ уезжают в MoonTrader сразу. Применяются разом по кнопке **💣 Деплой**
  (один перезапуск MT на все изменения). **↻ Перезагрузить** — взять актуальный конфиг
  из ядра и сбросить несохранённые изменения (с подтверждением). В шапке вкладки —
  индикатор «● Есть незадеплоенные изменения».
- **Чекбоксы выбора в Шаблонах** — квадратик у каждого алгоритма, как во вкладке «Алгоритмы».
- **Замочек приватности 🔒 запоминается** между вкладками и перезапусками.
- **Панель деплоя в Шаблонах** — сворачивается/разворачивается, аккаунты сгруппированы
  по биржам, у каждой биржи — «Выбрать все / Снять все».
- Значок деплоя везде — теперь 💣. Кнопка сохранения правок в редакторе — «💾 Сохранить (локально)».

### Исправления
- **«Не отвечает» у случайных аккаунтов** при проверке версий. Terminal опрашивал все
  сервера одним залпом с жёстким таймаутом 4 с — при большом парке часть рукопожатий
  случайно не успевала, и сервера «рандомно» помечались как недоступные. Теперь опрос
  идёт пулом (по 8 одновременно), таймаут 8 с и один автоповтор для не ответивших —
  связь определяется стабильно.

## v1.0.3 (2026-06-24)

Управление алгоритмами прямо с аккаунта (вкладка «Алгоритмы»).

### Новое
- **Создание алгоритмов и папок прямо на аккаунте** — кнопки «+ Алго» и «+ Папка»
  во вкладке «Алгоритмы» (как в Шаблонах). Можно быстро собрать один алгоритм и сразу
  задеплоить, не заводя шаблон. У каждой папки — «+» (добавить алго сюда) и «×» (удалить
  папку). Создание/удаление сразу применяется на сервере (MoonTrader перезапускается).
- **Удаление алгоритмов** — по одному (кнопка × у алгоритма) или сразу нескольких.
  Выбор нескольких — **квадратными чекбоксами** у каждого алгоритма (или Ctrl+клик) →
  «🗑 Удалить». С подтверждением: конфиг перезаписывается на сервере, MoonTrader
  перезапускается, старый конфиг бэкапится.
- **Перемещение в другую папку** — кнопка ⇄ у алгоритма → выбрать папку (или «без папки»).
- **Сортировка списка** — как в MT / по имени / по значению / по типу / по стороне,
  со стрелкой направления ↑↓. Сравнение натуральное (SELL 2 идёт раньше SELL 10).
  Только отображение в терминале — порядок в MoonTrader не меняется.
- **Сортировка по умолчанию** — в Настройках → «Отображение»: задаёшь один раз,
  применяется ко всем аккаунтам.

## v1.0.2 (2026-06-24)

Исправления корректности деплоя и стабильности перезапуска ядра.

### Что починили
- **Деплой на OKX и Bybit (и на ARM-серверах) теперь надёжный.** Раньше при деплое
  перезапуск MoonTrader иногда не срабатывал и приходила ложная ошибка
  «MTCore failed to start», хотя конфиг на самом деле применялся. Перезапуск ядра
  переписан — ядро корректно поднимается, ложных ошибок больше нет.
- **Сторона SELL теперь деплоится правильно.** Раньше SELL-алгоритм уезжал в MoonTrader
  с пустой стороной (не выбрано ни Buy, ни Sell) — MT не понимал значение. BUY работал
  всегда, поэтому проблема была только на SELL. Теперь корректны и деплой SELL, и
  отображение стороны при чтении из MT.
  > ⚠️ **SELL-шаблоны, созданные в прошлых версиях, нужно пересоздать** — или открыть
  > в редакторе, заново выбрать SELL и сохранить. В старых зашито неверное значение.
- **Тип тейк-профита (Classic / Historic / Depth)** — раньше «Classic» сохранялся
  неправильно, а «Historic» вообще нельзя было выбрать. Исправлено.
- **Подпись нового алгоритма в шаблонах** — теперь сразу показывает выбранные сторону
  и значение (раньше в списке могло оставаться «BUY 1.00» независимо от выбора).

> После обновления Terminal обновите ядро на серверах кнопкой ⬆ в карточке аккаунта
> (или «Обновить все»).

## v1.0.0 (2026-06-23) 🎉

Первый стабильный релиз — полноценный терминал для удалённого управления
MoonTrader-алгоритмами на парке серверов.

### Управление серверами

- **Авто-логи runtime-ошибок** React: `window.onerror`, `unhandledrejection`, `console.error` — всё уходит в `logs/<сегодня>.log`

- `restart_mt.sh`: **`pgrep -x MTCore`** вместо `pgrep -f` — раньше не находил процесс запущенный с относительным путём `./MTCore`

- `mt_service.py`: при ошибке скрипта возвращает stdout если stderr пуст + exit code (раньше «Restart failed:» был пустым)
