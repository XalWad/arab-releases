# ARAB — Changelog

## v0.8.0 (2026-05-19)

### Автообновления
- Проверка обновлений раз в час из публичного репо `XalWad/arab-releases`
- Зелёный бэйдж в шапке Terminal при наличии новой версии
- Модалка обновления с описанием и выбором: Terminal / Core / всё вместе
- Core: новый action `update_core` + скрипт `update.sh` + автоматический рестарт сервиса
- sudoers entry автоматически добавляется при первой установке Core

### Логирование
- Terminal: файл-логи в папке `logs/` рядом с .exe, ротация 7 дней
- Логируются все действия пользователя: смена вкладок, аккаунтов, CRUD шаблонов, deploy
- Core: файл-логи в `~/ARAB/logs/`, ротация 7 дней (плюс journalctl)
- Audit events: WS connections, deploy events с размерами/таймингами

### Шаблоны
- Inline-редактирование имени группы
- Папки с «авто-типом» (определяется первым добавленным алго)
- Кнопка `+ Алго` в общем header — создание без папки
- Перенос алгоритма между группами с фильтром совместимости

### Аккаунты
- Dropdown переключения аккаунтов прямо из шапки
- Авто-загрузка конфигурации после подключения

### Bulk Deploy
- Параллельный деплой шаблона на N серверов через отдельные WS-соединения
- Per-account статусы с retry на упавших
- Confirm-диалог перед массовой перезаписью

## v0.7.0 (2026-05-18)

### Templates tab
- Создание/редактирование/импорт/экспорт шаблонов конфигов алгоритмов
- Wizard создания алгоритма с нуля (MoonShot / DepthShot)
- Группировка алгоритмов по папкам

### Multi-account
- Поддержка нескольких аккаунтов ARAB Core
- Авто-определение MT-папки на стороне Core
- Передача `account_folder` в handshake

### Clone-as-template
- Кнопка «📋 → Шаблон» в Algorithms tab для копирования всего конфига в шаблон
