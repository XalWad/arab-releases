# ARAB — Changelog

## v1.0.3 (2026-06-24)

Управление алгоритмами прямо с аккаунта (вкладка «Алгоритмы»).

### Новое
- **Создание алгоритмов и папок прямо на аккаунте** — кнопки «+ Алго» и «+ Папка»
  во вкладке «Алгоритмы» (как в Шаблонах). Можно быстро собрать один алгоритм и сразу
  задеплоить, не заводя шаблон. У каждой папки — «+» (добавить алго сюда) и «×» (удалить
  папку). Создание/удаление сразу применяется на сервере (MoonTrader перезапускается).
- **Удаление алгоритмов** — по одному (кнопка × у алгоритма) или сразу нескольких.
  Выбор нескольких — **квадратными чекбоксами** у каждого алгоритма (или Ctrl+клик) →
  «🗑 Удалить». С подтверждением: конфиг перезаписывается на сервере, MoonTrader
  перезапускается, старый конфиг бэкапится.
- **Перемещение в другую папку** — кнопка ⇄ у алгоритма → выбрать папку (или «без папки»).
- **Сортировка списка** — как в MT / по имени / по значению / по типу / по стороне,
  со стрелкой направления ↑↓. Сравнение натуральное (SELL 2 идёт раньше SELL 10).
  Только отображение в терминале — порядок в MoonTrader не меняется.
- **Сортировка по умолчанию** — в Настройках → «Отображение»: задаёшь один раз,
  применяется ко всем аккаунтам.

## v1.0.2 (2026-06-24)

Исправления корректности деплоя и стабильности перезапуска ядра.

### Что починили
- **Деплой на OKX и Bybit (и на ARM-серверах) теперь надёжный.** Раньше при деплое
  перезапуск MoonTrader иногда не срабатывал и приходила ложная ошибка
  «MTCore failed to start», хотя конфиг на самом деле применялся. Перезапуск ядра
  переписан — ядро корректно поднимается, ложных ошибок больше нет.
- **Сторона SELL теперь деплоится правильно.** Раньше SELL-алгоритм уезжал в MoonTrader
  с пустой стороной (не выбрано ни Buy, ни Sell) — MT не понимал значение. BUY работал
  всегда, поэтому проблема была только на SELL. Теперь корректны и деплой SELL, и
  отображение стороны при чтении из MT.
  > ⚠️ **SELL-шаблоны, созданные в прошлых версиях, нужно пересоздать** — или открыть
  > в редакторе, заново выбрать SELL и сохранить. В старых зашито неверное значение.
- **Тип тейк-профита (Classic / Historic / Depth)** — раньше «Classic» сохранялся
  неправильно, а «Historic» вообще нельзя было выбрать. Исправлено.
- **Подпись нового алгоритма в шаблонах** — теперь сразу показывает выбранные сторону
  и значение (раньше в списке могло оставаться «BUY 1.00» независимо от выбора).

> После обновления Terminal обновите ядро на серверах кнопкой ⬆ в карточке аккаунта
> (или «Обновить все»).

## v0.9.9 (2026-06-23)

### Установка и серверы
- **Поддержка Ubuntu 26.04 / Python 3.14 (ARM)** — зависимости-диапазоны вместо
  жёстких пинов + установщик ставит `build-essential` и предпочитает стабильный
  Python, где он есть (фикс «pydantic не собирается / cc not found»)
- **Ядро больше не падает по кругу**, если в MT нет данных/алгоритмов — стартует
  штатно и спокойно ждёт конфиг
- **Анти-спам крэш-алертов** (не чаще раза в 2 мин) + **публичный IP** в алерте,
  чтобы сразу опознать упавший сервер

### Пустой MT-аккаунт
- Если в аккаунте ещё нет алгоритмов — вместо ошибки плашка с двумя путями:
  создать алгоритм в MoonTrader (**↻ Загрузить**) или **залить шаблон** в аккаунт
  (ARAB сам создаст `algorithms.config` и перезапустит MT)

### Настройки — внешний вид
- Новый вид карточки: версия Core справа, папка и адрес в одну строку, единый
  моноширинный шрифт и размер
- **«Горячая» подсветка**: CPU > 20% и RAM ≥ 85% — оранжевым
- IP и средние нагрузки (5м/1ч) сделаны ярче
- Форма добавления аккаунта — **над списком** (не нужно скроллить)
- **CSV-импорт по формату `Name,IP,Secret`** (порт подставляется сам)
- Блок «Telegram chat ID для всех» свёрнут; раскрывашки (📗 CSV / 🔔 Telegram)
  в едином стиле

### Исправления
- **Надёжная проверка обновлений** (без кэша + перепроверка по кнопке «↻ Статус
  серверов») — бэйдж новой версии появляется сразу, без перезапуска
- **Нет ложного 🔴** при обновлении ядра — обрыв связи больше не считается ошибкой

## v0.9.6 (2026-06-19)

### Установка ядра «в одну команду»
- **Новый установщик `bootstrap.sh`** — ядро ставится на сервер одной командой:
  сам доустанавливает окружение, скачивает код, находит MoonTrader, генерирует
  секрет, настраивает автозапуск, открывает порт и печатает адрес+секрет для Terminal
- **Telegram-бот владельца «впиши один раз»** (`owner-telegram.conf`) — токен и
  admin chat_id вписываются однократно и сами подставляются на каждый новый сервер
- Инструкция «для чайника» `QUICK-INSTALL.md`; `INSTALL.md` переписан под новый способ

### Порт
- **Стандартный порт ARAB → 4343** (был 8765). Новые сервера сразу на 4343;
  порт читается из `.env` (можно переопределить через `export ARAB_PORT=...`)

### Terminal UI
- **В форме аккаунта вписывается только IP** — края `ws://` … `:4343/ws`
  зафиксированы, порт по умолчанию 4343. Вставка целого адреса тоже понимается

## v0.9.0 (2026-05-20)

### Авто-обновления
- **electron-updater**: автоустановка Terminal в фоне (прогресс-бар → перезапуск → готово). Больше не нужно качать .exe вручную
- **Параллельный fan-out `update_core`** на N серверов с per-account статусами (как bulk deploy)
- **Авто-probe версий Core** при открытии модалки обновлений: версии всех аккаунтов автоматически опрашиваются через короткие WS, не нужно переключаться по очереди. Плюс кнопка «↻ Перепроверить версии»
- **Полная автоматизация публикации релизов**: `pwsh release.ps1 -Version X.Y.Z` создаёт GitHub Release, заливает `.exe` + `latest.yml` + `.blockmap`, обновляет `VERSION.json` в публичном репо — одной командой
- `.bat`-файлы в корне репо для запуска без знания cmd: `release.bat`, `release-dryrun.bat`, `start-terminal.bat`
- Валидация semver в `release.ps1` — не даёт случайно вписать `0.9.01` (ведущий ноль ломает electron-updater)

### Templates UX
- **Resizable панели** (drag-handle) во вкладках Алгоритмы и Шаблоны. Ширина каждой панели запоминается в localStorage
- **Импорт алго из шаблона в живой конфиг** (кнопка `← Шаблон` в Algorithms tab). Опции: «добавить как новые» или «заменить по ID». Группы шаблона, на которые ссылаются алго, автоматически переносятся
- **Бэйдж «● несохранено»** в шаблонах + защита от потери изменений при переключении на другой алго / шаблон
- **CSV-импорт аккаунтов** (формат: `name,url,secret` построчно, с поддержкой `"кавычек"` и `#` комментариев)

### История изменений + Rollback
- Новая вкладка **«История»** с фильтрами по типу/аккаунту (до 200 записей в localStorage)
- Каждое действие (single-deploy, bulk-deploy, template-deploy, template-import, rollback) автоматически логируется
- Кнопка **«↶ Откатить»** на любой успешной записи → модалка `BackupsModal` с выбором `.bak` файла на сервере
- Core: retention бэкапов увеличен с 5 до **30**, новые WS-actions `list_backups` + `restore_backup`
- Перед откатом Core автоматически бэкапит текущее состояние — на случай отмены отката

### Logging & Diagnostics
- **Авто-логи runtime-ошибок** React: `window.onerror`, `unhandledrejection`, `console.error` — всё уходит в `logs/<сегодня>.log`
- **React ErrorBoundary**: при crash рендера показывает дружелюбный экран с ошибкой и кнопкой «Открыть логи»
- **Кнопка «📂 Открыть»** в Settings → Диагностика, открывает папку логов в проводнике

### Фиксы
- `restart_mt.sh`: **`pgrep -x MTCore`** вместо `pgrep -f` — раньше не находил процесс запущенный с относительным путём `./MTCore`
- `restart_mt.sh`: **polling до 15 сек** вместо одной проверки через 2 сек после fallback-запуска
- `mt_service.py`: при ошибке скрипта возвращает stdout если stderr пуст + exit code (раньше «Restart failed:» был пустым)
- **`tsc --noEmit`** теперь часть основного `npm run build` — TS-ошибки ловятся на стадии сборки, а не в runtime у пользователя
