# ARAB — Changelog

## v0.9.0 (2026-05-20)

### Авто-обновления
- **electron-updater**: автоустановка Terminal в фоне (прогресс-бар → перезапуск → готово). Больше не нужно качать .exe вручную
- **Параллельный fan-out `update_core`** на N серверов с per-account статусами (как bulk deploy)
- **Авто-probe версий Core** при открытии модалки обновлений: версии всех аккаунтов автоматически опрашиваются через короткие WS, не нужно переключаться по очереди. Плюс кнопка «↻ Перепроверить версии»
- **Полная автоматизация публикации релизов**: `pwsh release.ps1 -Version X.Y.Z` создаёт GitHub Release, заливает `.exe` + `latest.yml` + `.blockmap`, обновляет `VERSION.json` в публичном репо — одной командой
- `.bat`-файлы в корне репо для запуска без знания cmd: `release.bat`, `release-dryrun.bat`, `start-terminal.bat`
- Валидация semver в `release.ps1` — не даёт случайно вписать `0.9.01` (ведущий ноль ломает electron-updater)

### Templates UX
- **Resizable панели** (drag-handle) во вкладках Алгоритмы и Шаблоны. Ширина каждой панели запоминается в localStorage
- **Импорт алго из шаблона в живой конфиг** (кнопка `← Шаблон` в Algorithms tab). Опции: «добавить как новые» или «заменить по ID». Группы шаблона, на которые ссылаются алго, автоматически переносятся
- **Бэйдж «● несохранено»** в шаблонах + защита от потери изменений при переключении на другой алго / шаблон
- **CSV-импорт аккаунтов** (формат: `name,url,secret` построчно, с поддержкой `"кавычек"` и `#` комментариев)

### История изменений + Rollback
- Новая вкладка **«История»** с фильтрами по типу/аккаунту (до 200 записей в localStorage)
- Каждое действие (single-deploy, bulk-deploy, template-deploy, template-import, rollback) автоматически логируется
- Кнопка **«↶ Откатить»** на любой успешной записи → модалка `BackupsModal` с выбором `.bak` файла на сервере
- Core: retention бэкапов увеличен с 5 до **30**, новые WS-actions `list_backups` + `restore_backup`
- Перед откатом Core автоматически бэкапит текущее состояние — на случай отмены отката

### Logging & Diagnostics
- **Авто-логи runtime-ошибок** React: `window.onerror`, `unhandledrejection`, `console.error` — всё уходит в `logs/<сегодня>.log`
- **React ErrorBoundary**: при crash рендера показывает дружелюбный экран с ошибкой и кнопкой «Открыть логи»
- **Кнопка «📂 Открыть»** в Settings → Диагностика, открывает папку логов в проводнике

### Фиксы
- `restart_mt.sh`: **`pgrep -x MTCore`** вместо `pgrep -f` — раньше не находил процесс запущенный с относительным путём `./MTCore`
- `restart_mt.sh`: **polling до 15 сек** вместо одной проверки через 2 сек после fallback-запуска
- `mt_service.py`: при ошибке скрипта возвращает stdout если stderr пуст + exit code (раньше «Restart failed:» был пустым)
- **`tsc --noEmit`** теперь часть основного `npm run build` — TS-ошибки ловятся на стадии сборки, а не в runtime у пользователя

## v0.8.0 (2026-05-19)

### Автообновления
- Проверка обновлений раз в час из публичного репо `XalWad/arab-releases`
- Зелёный бэйдж в шапке Terminal при наличии новой версии
- Модалка обновления с описанием и выбором: Terminal / Core / всё вместе
- Core: новый action `update_core` + скрипт `update.sh` + автоматический рестарт сервиса
- sudoers entry автоматически добавляется при первой установке Core

### Логирование
- Terminal: файл-логи в папке `logs/` рядом с .exe, ротация 7 дней
- Логируются все действия пользователя: смена вкладок, аккаунтов, CRUD шаблонов, deploy
- Core: файл-логи в `~/ARAB/logs/`, ротация 7 дней (плюс journalctl)
- Audit events: WS connections, deploy events с размерами/таймингами

### Шаблоны
- Inline-редактирование имени группы
- Папки с «авто-типом» (определяется первым добавленным алго)
- Кнопка `+ Алго` в общем header — создание без папки
- Перенос алгоритма между группами с фильтром совместимости

### Аккаунты
- Dropdown переключения аккаунтов прямо из шапки
- Авто-загрузка конфигурации после подключения

### Bulk Deploy
- Параллельный деплой шаблона на N серверов через отдельные WS-соединения
- Per-account статусы с retry на упавших
- Confirm-диалог перед массовой перезаписью
